## 4. 实验操作标准化流程 (SOP)

### 第一步：精度评估 (Evaluation)
**工具**：`eval_mini_official.py`
**关键路径**：`work_dirs/teacher_eval/results/pred_instances_3d/results_nusc.json`
**注意**：官方权重推理时需在 Slurm 脚本中建立 `v1.0-mini` 到 `v1.0-trainval` 的软链接，否则评估器会因找不到数据集目录而报错。

### 第二步：算力统计 (FLOPs & Params)
**工具**：`calc_student_metrics.py` (Hook 挂载法)
**逻辑**：手动补全 `data_preprocessor`，通过前向传播 Hook 动态统计卷积核运算次数。
**结果**：证实手动剪枝（Backbone/Neck 通道减半）成功将算力从 115.94G 降至 91.25G。

### 第三步：速度测试 (Benchmark FPS)
**工具**：`tools/analysis_tools/benchmark.py`
**结论**：3D 检测中存在由体素化 (Voxelization) 和后处理 (NMS) 导致的固定耗时，模型参数减小对 FPS 提升具有“边际效用递减”现象。

---

## 5. 核心结论 (Final Analysis)
1. **蒸馏优越性**：同架构小模型 (91G) 虽然参数更少，但其 NDS (**0.2439**) 显著优于同模蒸馏 (**0.2149**)。证明在有限的小规模数据集（Mini Set）下，**精准剪枝后的模型由于参数冗余更少，反而更容易通过蒸馏学习到有效的特征表达。**
2. **性能保留率**：尽管官方 Teacher 拥有 300 倍于学生的训练数据量，但 91G 学生模型在 **Car** 和 **Pedestrian** 等核心类别上依然分别保留了状元 **69%** 和 **82.5%** 的精度，证明了该轻量化方案的工程实用价值。
